<h3>A01 - Broken Access Control</h3>
- Access control enforces polciy that outside users cant act outside the permissions granted for them
<br>
- failure in this -> unauthorized access, modifications etc.
<br>
- Bypassing the access control can be done by modifying the URL,
internal application state or HTML page, or tool modifying APIs
<br>
- API access POST, PUT and DELETE
<br>
Prevention: <br>
-by handling it on trusted server <br>
- default deny-rules <br>
- ownership-based data restrictions <br>
- secure server configs. <br>
- session/tokem lifecycle management <br>

<h3>Access control vulnerabilities and privilege escalation</h3> <br>
- Access control -> whether user is allowed to carry the action <br>
- broken control is common -> critical security vulnerability <br>
- Vertical = restrict access to sensitive functionality to specific type users <br>
- Horizontal = restrict access to specific users <br>
- Context-dependent = restriction based on the state of application or users interaction <br>
- Parameter-based = storing info at login like hidden field, cookie, query string parameter <br>
