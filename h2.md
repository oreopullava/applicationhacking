<h3>A01 - Broken Access Control</h3>
- Access control enforces polciy that outside users cant act outside the permissions 
granted for them
- failure in this -> unauthorized access, modifications etc.
- Bypassing the access control can be done by modifying the URL,
internal application state or HTML page, or tool modifying APIs
- API access POST, PUT and DELETE
Prevention:
-by handling it on trusted server
- default deny-rules
- ownership-based data restrictions
- secure server configs.
- session/tokem lifecycle management

<h3>Access control vulnerabilities and privilege escalation</h3>
- Access control -> whether user is allowed to carry the action
- broken control is common -> critical security vulnerability
- Vertical = restrict access to sensitive functionality to specific type users
- Horizontal = restrict access to specific users
- Context-dependent = restriction based on the state of application or users interaction
- Parameter-based = storing info at login like hidden field, cookie, query string parameter
